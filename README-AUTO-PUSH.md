# 자동 푸시 설정 가이드

이 프로젝트에 파일 변경 시 자동으로 GitHub에 푸시하는 기능을 설정할 수 있습니다.

## 방법 1: Git Hook 사용 (권장)

Git hook을 사용하면 커밋할 때마다 자동으로 푸시됩니다.

### 설정 방법

1. `.git/hooks/post-commit` 파일이 생성되었는지 확인
2. Windows에서는 Git Bash를 사용하여 커밋하거나, 아래 PowerShell 스크립트 사용

### 사용 방법

```bash
# 일반적인 커밋
git add .
git commit -m "변경사항 설명"
# 자동으로 푸시됨
```

## 방법 2: 파일 감시 스크립트 사용

파일을 저장할 때마다 자동으로 커밋하고 푸시합니다.

### 사용 방법

1. PowerShell에서 실행:
```powershell
.\watch-and-push.ps1
```

2. 이제 파일을 저장하면 자동으로 커밋되고 푸시됩니다.
3. 종료하려면 `Ctrl+C`

### 주의사항

- `.git`, `node_modules`, `.vscode`, `.cursor` 폴더는 제외됩니다
- 파일 저장 후 5초 대기 후 커밋됩니다 (디바운싱)
- 커밋 메시지는 자동으로 생성됩니다

## 방법 3: 수동 실행 스크립트

필요할 때만 실행하는 스크립트:

```powershell
.\auto-push.ps1
```

## 권장 설정

**VS Code 사용자:**
- VS Code의 Git 확장 기능 사용
- 커밋 시 자동 푸시 옵션 활성화

**자동화를 원하는 경우:**
- `watch-and-push.ps1` 스크립트를 백그라운드에서 실행
- 또는 작업 스케줄러에 등록

## 주의사항

⚠️ **자동 푸시는 편리하지만 주의가 필요합니다:**
- 민감한 정보가 커밋되지 않도록 주의
- `.gitignore` 파일을 올바르게 설정
- 테스트 코드나 불완전한 코드가 푸시될 수 있음
